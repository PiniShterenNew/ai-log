# AI Question Log

משימת בית - ממשק לניהול וצפייה בשאלות AI שנשאלו על ידי לקוחות במערכת, כולל התשובות ומדדי ביצועים.

## מה הפרויקט עושה

האפליקציה מציגה ממשק לניהול שאלות AI עם שתי דפים:

### 1. דף רשימת שאלות
- **טבלת שאלות** - תצוגה מקדימה של שאלות, תאריך, זמן תגובה, ומזהה משתמש
- **סינון לפי תאריך** - סינון שאלות בטווח תאריכים
- **מיון** - מיון לפי תאריך (עולה/יורד)

### 2. דף פרטי שאלה
- **פרטים מלאים** - שאלה מלאה, תשובת AI, משתמש, תאריך, וזמן תגובה
- **ניווט חזרה** - חזרה לרשימה תוך שמירה על מצב הסינון והמיון

## מבנה הפרויקט

הפרויקט בנוי עם הפרדה ברורה בין שכבות:

```
src/
├── features/           # Features לפי domain
│   ├── QuestionsList/ # דף רשימת שאלות
│   └── QuestionItem/  # דף פרטי שאלה
├── design-system/     # ספריית קומפוננטות גנריות (Design System)
│   └── components/    # קומפוננטות לשימוש חוזר (layout, typography, controls)
├── domain/            # לוגיקה עסקית (סינון, מיון)
├── data/              # Repository layer + mock data
└── shared/            # Utilities משותפים (פורמט תאריכים, משך זמן)
```

**Design System** - ספריית קומפוננטות נפרדת המכילה קומפוננטות גנריות לשימוש חוזר, מאורגנות לפי קטגוריות (layout, typography, controls, feedback וכו').

## טכנולוגיות

- **React 19** + **TypeScript**
- **Vite** - Build tool
- **React Router** - ניתוב
- **Tailwind CSS** - עיצוב
- **Vitest** - בדיקות
- **Radix UI** - קומפוננטות בסיס

## Mock Data

הפרויקט משתמש ב-mock data סטטי המכיל:
- שאלות AI מגוונות
- תשובות AI מלאות
- משתמשים שונים
- תאריכים וזמני תגובה מגוונים

הנתונים מדמים תרחישים שונים ומאפשרים בדיקה של כל הפונקציונליות.

## הנחות יסוד

- כל הנתונים הם mock data (אין backend אמיתי)
- תאריכים נשמרים ב-UTC ומוצגים בזמן מקומי
- משך זמן תגובה מוצג במילישניות (<1 שנייה) או בשניות (≥1 שנייה)
- מצב הסינון והמיון נשמר ב-URL query params
